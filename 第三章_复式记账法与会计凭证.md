## 一、复式记账法原理

### 1.1 定义

每一笔经济业务都在**两个或两个以上**账户中相互关联地记录。

### 1.2 核心规则

**有借必有贷，借贷必相等**

### 1.3 为什么叫"复式"？

| 记账方式 | 记录数量 | 示例 |
|----------|----------|------|
| 单式记账 | 只记一笔 | 付现金1000元 |
| **复式记账** | 同时记两笔以上 | 资产↓ + 费用↑ |

### 1.4 示例：用银行存款购买设备

```
业务：用银行存款50000元购买设备

借：固定资产    50000
贷：银行存款    50000
```

**分析：**
- 固定资产增加（资产在借方）
- 银行存款减少（资产在贷方）
- 等式仍然平衡：资产不变

## 三、会计凭证类型

### 3.1 原始凭证

| 类型 | 说明 | 举例 |
|------|------|------|
| 外来原始凭证 | 从外部取得的凭证 | 发票、收据 |
| 自制原始凭证 | 企业内部编制的凭证 | 领料单、工资表 |

### 3.2 记账凭证

| 类型 | 用途 |
|------|------|
| 收款凭证 | 记录现金/银行存款收入 |
| 付款凭证 | 记录现金/银行存款支出 |
| 转账凭证 | 记录不涉及现金收付的业务 |

## 四、实操：一笔完整业务的分录

### 业务案例

**1月15日，从工商银行提取现金5000元备用。**

### 分录

```
借：库存现金          5,000.00
贷：银行存款-工商银行  5,000.00
```

### 解释

| 账户 | 方向 | 金额 | 原因 |
|------|------|------|------|
| 库存现金 | 借 | 5,000 | 资产增加 |
| 银行存款-工商银行 | 贷 | 5,000 | 资产减少 |

### T型账户展示

```
库存现金              银行存款
----------            ----------
借     贷            借     贷
------ 5,000
```

## 五、MySQL表设计扩展

### 5.1 凭证主表

```sql
CREATE TABLE accounting_vouchers (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '凭证ID',
    voucher_no VARCHAR(20) NOT NULL UNIQUE COMMENT '凭证号',
    voucher_date DATE NOT NULL COMMENT '凭证日期',
    summary VARCHAR(200) COMMENT '摘要',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会计凭证表';
```

### 5.2 凭证明细表（复式记账需要双方记录）

```sql
CREATE TABLE accounting_voucher_items (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '明细ID',
    voucher_id INT NOT NULL COMMENT '凭证ID（外键）',
    direction ENUM('借','贷') NOT NULL COMMENT '借贷方向',
    subject_code VARCHAR(10) NOT NULL COMMENT '科目代码',
    amount DECIMAL(12,2) NOT NULL COMMENT '金额',
    FOREIGN KEY (voucher_id) REFERENCES accounting_vouchers(id),
    FOREIGN KEY (subject_code) REFERENCES accounting_subjects(subject_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='凭证明细表';
```

### 5.3 插入示例数据

```sql
-- 凭证1：提取现金
INSERT INTO accounting_vouchers (voucher_no, voucher_date, summary)
VALUES ('记-001', '2026-01-15', '从工商银行提取现金备用');

INSERT INTO accounting_voucher_items (voucher_id, direction, subject_code, amount)
VALUES 
    (1, '借', '1001', 5000.00),      -- 库存现金
    (1, '贷', '1002', 5000.00);     -- 银行存款-工商银行
```

### 5.4 验证借贷平衡

```sql
-- 检查借贷是否平衡
SELECT 
    voucher_id,
    SUM(CASE WHEN direction='借' THEN amount ELSE 0 END) AS debit_total,
    SUM(CASE WHEN direction='贷' THEN amount ELSE 0 END) AS credit_total
FROM accounting_voucher_items
GROUP BY voucher_id
HAVING debit_total != credit_total;
-- 如果无结果，说明平衡
```
