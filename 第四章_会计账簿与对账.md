## 一、会计账簿概述

### 1.1 账簿的定义

**会计账簿**是以会计凭证为依据，全面、连续、系统地记录经济业务的簿籍。

| 组件 | 说明 |
|------|------|
| 账页 | 账簿的具体记录载体 |
| 账本 | 若干账页装订在一起 |

### 1.2 账簿的作用

1. 分类汇总经济业务
2. 反映资产、负债、权益变动
3. 为报表提供数据
4. 进行会计监督的依据

---

## 二、账簿的种类

### 2.1 按用途分类

| 类型 | 说明 | 举例 |
|------|------|------|
| **序时账簿** | 按时间顺序记录 | 日记账、银行存款日记账 |
| **分类账簿** | 按科目分类记录 | 总账、明细账 |
| **备查账簿** | 补充记录 | 代管商品、租入固定资产 |

### 2.2 按外表形式分类

| 类型 | 特点 |
|------|------|
| 订本式 | 固定装订，防止抽换 |
| 活页式 | 灵活增减，便于分工 |
| 卡片式 | 长期使用，大型资产 |

### 2.3 账簿格式

**三栏式账簿**
```
┌─────────────────────────────┐
│ 账户名称：银行存款           │
├─────┬───────┬───────┬────────┤
│ 日期 │ 摘要  │ 借方  │ 贷方   │
├─────┼───────┼───────┼────────┤
│     │       │       │        │
└─────┴───────┴───────┴────────┘
```

**数量金额式账簿**（适用于存货）
```
┌──────────────────────────────────────┐
│ 账户名称：原材料-甲材料              │
├──────┬────┬──────┬────┬──────┬──────┤
│ 日期 │凭证│ 摘要 │数量│ 单价 │ 金额 │
├──────┼────┼──────┼────┼──────┼──────┤
│      │    │      │    │      │      │
└──────┴────┴──────┴────┴──────┴──────┘
```

## 三、登记账簿的规则

### 3.1 基本规则

| 规则 | 说明 |
|------|------|
| 记账依据 | 审核无误的会计凭证 |
| 记账符号 | 借方记左，贷方记右 |
| 连续记录 | 不得跳行、隔页 |
| 及时登记 | 当日业务当日记 |

### 3.2 记账要素

1. **日期**：业务发生时间
2. **凭证号**：记账凭证编号
3. **摘要**：业务简要说明
4. **金额**：借贷方向和金额

### 3.3 日记账格式示例

**银行存款日记账（丁字账）**

```
银行存款
═══════════════════════════════
借方                贷方
───────────────────────────────
期初余额  100,000    提取现金   5,000
收到货款   50,000    购设备    30,000
                  支付费用    2,000
───────────────────────────────
本期发生  50,000    本期发生  37,000
期末余额 113,000
```

## 四、对账

### 4.1 对账的意义

**对账**是确保账簿记录与实际相符的过程。

### 4.2 对账内容

| 类型 | 核对内容 |
|------|----------|
| **账证核对** | 账簿记录与凭证核对 |
| **账账核对** | 总账与明细账核对 |
| **账实核对** | 账簿与实物核对（现金、存货） |
| **账表核对** | 账簿与报表核对 |

### 4.3 对账流程图

```
原始凭证 → 记账凭证 → 账簿记录
                ↓
          账证核对（是否一致）
                ↓
          账账核对（总账vs明细账）
                ↓
          账实核对（实物盘点）
                ↓
            完成对账
```

## 五、错账更正方法

### 5.1 划线更正法

**适用：记账后发现账簿文字或数字有错误**

```
更正方法：在错误文字/数字上划红线，写上正确数字
举例：
  原记：银行存款    1,500
  更正：银行存款    1,600
       ↑划线更正
```

### 5.2 红字更正法

**适用：记账后发现记账方向或科目错误**

```
更正步骤：
1. 用红字编制相同的错误分录
2. 用蓝字编制正确分录
举例：
  原错误分录：
    借：管理费用  1,500
    贷：银行存款  1,500
  
  红字冲销：
    借：管理费用  [1,500]  ←红字
    贷：银行存款  [1,500]  ←红字
  
  正确分录：
    借：财务费用    1,500
    贷：银行存款  1,500
```

### 5.3 补充登记法

**适用：记账后发现金额少记**

```
更正方法：补记少记金额
举例：
  原记录：借：应收账款  8,000
  实际应为：借：应收账款  9,000
  
  补充分录：
    借：应收账款  1,000
    贷：相关科目  1,000
```

## 六、结账

### 6.1 结账的定义

**结账**是在期末对账簿记录进行总结，计算出本期发生额和余额。

### 6.2 结账步骤

1. **结转收入** → 本年利润
2. **结转费用** → 本年利润
3. **结清临时性账户**（损益类）
4. **划线结出余额**

### 6.3 结账格式

```
                         月结                    年结
─────────────────┬─────────────────┬─────────────────┐
  本月发生额      │    本月余额     │  本年累计发生额 │
─────────────────┼─────────────────┼─────────────────┤
借方   │  贷方   │  借方  │  贷方  │  借方  │  贷方  │
───────┼─────────┼────────┼────────┼────────┼────────┤
       │         │        │        │        │        │
───────┴─────────┴────────┴────────┴────────┴────────┘
```

## 七、MySQL账簿表设计

### 7.1 账簿表

```sql
CREATE TABLE account_books (
    id INT AUTO_INCREMENT PRIMARY KEY,
    book_name VARCHAR(50) NOT NULL COMMENT '账簿名称',
    book_type ENUM('日记账','分类账','备查账') NOT NULL COMMENT '账簿类型',
    subject_code VARCHAR(10) COMMENT '关联科目',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 7.2 账簿记录表

```sql
CREATE TABLE book_entries (
    id INT AUTO_INCREMENT PRIMARY KEY,
    book_id INT NOT NULL COMMENT '账簿ID',
    entry_date DATE NOT NULL COMMENT '记账日期',
    voucher_id INT COMMENT '凭证ID',
    summary VARCHAR(200) COMMENT '摘要',
    debit DECIMAL(12,2) DEFAULT 0 COMMENT '借方金额',
    credit DECIMAL(12,2) DEFAULT 0 COMMENT '贷方金额',
    balance DECIMAL(12,2) COMMENT '余额',
    entry_type ENUM('期初','本期','期末') DEFAULT '本期',
    FOREIGN KEY (book_id) REFERENCES account_books(id),
    FOREIGN KEY (voucher_id) REFERENCES accounting_vouchers(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 7.3 对账记录表

```sql
CREATE TABLE reconciliation (
    id INT AUTO_INCREMENT PRIMARY KEY,
    check_date DATE NOT NULL COMMENT '对账日期',
    book_id INT NOT NULL COMMENT '账簿ID',
    check_type ENUM('账证','账账','账实') NOT NULL COMMENT '对账类型',
    status ENUM('一致','不一致','待查') DEFAULT '一致',
    difference DECIMAL(12,2) DEFAULT 0 COMMENT '差异金额',
    note VARCHAR(500) COMMENT '备注',
    FOREIGN KEY (book_id) REFERENCES account_books(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
