## 一、会计科目的定义

**会计科目**是对会计要素的具体分类，是设置账户的依据。

| 术语 | 含义 |
|------|------|
| 科目 | 会计要素的具体分类名称 |
| 账户 | 具有格式的科目载体，用于记录经济业务 |

---

## 二、会计科目的分类

### 2.1 按经济内容分类

| 大类 | 科目举例 | 余额方向 |
|------|----------|----------|
| **资产类** | 银行存款、应收账款、固定资产 | 借方 |
| **负债类** | 应付账款、短期借款、应付票据 | 贷方 |
| **所有者权益类** | 实收资本、资本公积、盈余公积 | 贷方 |
| **成本类** | 生产成本、制造费用 | 借方 |
| **损益类** | 主营业务收入、管理费用 | 视情况 |

### 2.2 按提供信息详细程度

```
一级科目（总账科目）→ 二级科目（子目）→ 三级科目（明细科目）
   例：银行存款        → 工商银行        → 基本户
```

## 四、T型账户结构

```
┌─────────────────┬─────────────────┐
│       借方       │       贷方       │
├─────────────────┼─────────────────┤
│  资产增加        │  资产减少       │
│  负债减少        │  负债增加       │
│  所有者权益减少  │  所有者权益增加  │
│  成本增加        │  成本减少       │
│  费用增加        │  费用减少       │
│  收入减少        │  收入增加       │
├─────────────────┼─────────────────┤
│  期末余额（资产）│  期末余额（负债）│
└─────────────────┴─────────────────┘
```

**规则：有借必有贷，借贷必相等**

## 五、MySQL建表实践

### 5.1 会计科目表

```sql
CREATE TABLE accounting_subjects (
    subject_code VARCHAR(10) PRIMARY KEY COMMENT '科目编码',
    subject_name VARCHAR(50) NOT NULL COMMENT '科目名称',
    subject_type ENUM('资产','负债','所有者权益','成本','损益') NOT NULL COMMENT '科目类型',
    direction ENUM('借','贷') NOT NULL COMMENT '余额方向',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会计科目表';
```

### 5.2 插入示例数据

```sql
-- 资产类
INSERT INTO accounting_subjects VALUES ('1001','库存现金','资产','借');
INSERT INTO accounting_subjects VALUES ('1002','银行存款','资产','借');
INSERT INTO accounting_subjects VALUES ('1122','应收账款','资产','借');

-- 负债类
INSERT INTO accounting_subjects VALUES ('2001','短期借款','负债','贷');
INSERT INTO accounting_subjects VALUES ('2202','应付账款','负债','贷');

-- 所有者权益类
INSERT INTO accounting_subjects VALUES ('4001','实收资本','所有者权益','贷');
```

### 5.3 验证查询

```sql
-- 查询所有科目
SELECT subject_code, subject_name, subject_type, direction 
FROM accounting_subjects 
ORDER BY subject_type, subject_code;
```
